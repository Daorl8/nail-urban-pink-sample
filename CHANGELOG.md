## 2026-07-14 — 앵커 부드러운 이동 (rAF 커스텀)
- **증상**: 상단 메뉴 클릭 시 즉시 점프. **원인은 코드 누락이 아님** — `html{scroll-behavior:smooth;scroll-padding-top:74px}`는 처음부터 있었음.
- **진짜 원인**: OS '동작 줄이기(prefers-reduced-motion: reduce)'가 켜져 있으면 브라우저가 **native smooth scroll을 전부 즉시 이동으로 강제**한다. 실측(reduce=true 환경):
  | 방식 | 결과 |
  |---|---|
  | CSS `scroll-behavior:smooth` | 즉시 점프 |
  | `scrollTo({behavior:'smooth'})` | 즉시 점프 |
  | `scrollIntoView({behavior:'smooth'})` | 즉시 점프 |
  | rAF 직접 애니메이션 | 43프레임 = 작동 |
- **처리(사용자 판단 = 전원 강제)**: `a[href^="#"]` 위임 클릭 핸들러 + **rAF 커스텀 스크롤 700ms / easeInOutCubic**. OS 설정과 무관하게 모두 동일하게 동작. 루프 안에서는 `behavior:'instant'`로 CSS smooth와 충돌 방지.
  - ⚠️ 접근성 트레이드오프를 알고 내린 결정 — 어지러움·멀미로 설정을 켠 방문자의 의사를 거스름.
- **⚠️ 안전장치(중요)**: rAF는 **탭이 백그라운드면 아예 안 돈다**(실측: hidden 탭에서 500ms간 프레임 0). 그런데 핸들러가 이미 `preventDefault()`를 한 뒤라 **링크가 완전히 죽어버렸다.** → `setTimeout(DUR+300)` 가드 추가: 시간 초과 시 즉시 이동으로 폴백. 이동 같은 기본 동작을 rAF에만 의존시키면 안 됨.
- 부가: 스크롤 중 사용자가 휠·터치하면 애니메이션 즉시 양보 / 해시는 `replaceState`(뒤로가기 히스토리 오염 없음) / 도착 시 섹션에 `tabindex=-1` + `focus()`(키보드·스크린리더용, blur 시 tabindex 제거).
- 라이트박스 링크는 `href="nu-g1.jpg"`라 `^="#"`에 안 걸림 → 영향 없음(열림·ESC·alt 승계 재확인).
- **검증**: 보이는 탭 = 5개 링크 전부 40~43프레임 애니메이션 / 숨은 탭 = 가드 폴백으로 `0 → 5289` 정확 착지. 착지 위치 전부 헤더 아래 74~75px(헤더 65px → 여백 9~10px), 가림 없음.

## 2026-07-14 — 코드리뷰 2차: 스크롤 재진입 버그 수정
- **① 재진입 버그(실제 결함, 재현 완료)**: `busy`가 `go()` **바깥의 공유 불리언**인데 `t0`/`guard`/`done`/`step`은 클로저 지역이었음. 앵커를 연속 클릭하면 루프 두 개가 동시에 살아 서로 다른 목표로 `scrollTo` 경합 → 먼저 끝난 쪽이 `busy=false`로 나중 것을 죽임.
  - 실측 재현: 소개 클릭 → 200ms 뒤 갤러리 클릭 → **소개(702)에 멈추고 해시 `#about`**(갤러리 2788이어야 함). 50/200/500ms 간격 전부 재현.
  - 수정: **공유 `busy` 제거 → `seq` 소유권 토큰**. `go()` 진입 시 `var my=++seq`로 이전 루프 전부 무효화, `step`/`settle`/`guard`가 각자 `if(my!==seq) return`.
  - 부수: 휠 취소 시 **해시를 갱신하지 않도록** 변경(도착하지 않았는데 해시만 바뀌던 문제).
  - 재검증: 50/200/500ms 연속 클릭 + 3연속 클릭 + 단일 클릭 회귀 = **전부 마지막 목표에 정확 착지, 해시 일치**.
  - ⚠️ 리뷰의 "A의 guard가 살아있어 A 목표로 순간이동" 부분은 부정확 — A의 `done()`이 자기 guard를 clear했고, B의 guard는 `if(busy)`에 막혔음. 다만 **핵심 진단은 정확했고 버그는 실재**.
- **③ `.pl li>span em`**("기본·시럽·컬러", 12.5px): `--ink-faint` → **`--ink-soft`**. 흰 카드 대비 **3.52 → 5.45** AA 통과. (리뷰는 3.31로 계산했으나 실측 3.52 — 어느 쪽이든 4.5 미달이라 결론 동일.)

- **② reduced-motion 강제 → 재확인 후에도 유지**(사용자 판단, 2026-07-14 재확인). 리뷰가 새 근거 제시 — `.rv` 리빌은 `no-preference`로 게이팅하면서 스크롤만 rAF로 우회하는 건 방향이 반대이고(전정기관 트리거는 전체화면 700ms 스크롤 > 작은 페이드인), MDN·web.dev가 reduce에서 가장 먼저 끄라고 드는 예가 스크롤 애니메이션. **타당한 지적이나 사용자가 알고도 강제를 택함.**
  - ⚠️ **알려진 모순으로 남김**: `.rv`는 존중 / 스크롤은 강제. 되돌리려면 `go()` 앞 2줄(`matchMedia('(prefers-reduced-motion:reduce)').matches` → instant + settle) 이면 됨.

`.tagline`은 **사진 위**에 있는데, 내 스캔의 `bgOf()`는 부모를 타고 올라가 **불투명 배경색**을 찾는다 — 사진을 못 본다. 히어로의 `background:var(--bg2)`를 배경으로 잡고 통과 판정했다. **"텍스트 80개 전수 미달 0"을 여러 번 보고했으나, 사진 위 텍스트는 애초에 검사되지 않고 있었다.** STRUCTURE의 "알려진 미달 2건"이 맞고 CHANGELOG가 틀렸음.

### STRUCTURE 로즈 잔재 제거 (리뷰 지적 — 정확했음)
`index.html`에 `#C1467A`는 **0회**인데 STRUCTURE엔 살아있는 규칙처럼 남아 있었음. **토큰 값은 갱신했으나 토큰에 대한 규칙이 안 따라온 것.** 다올 규칙이 "작업 전 STRUCTURE 필독"이라 다음 세션을 적극적으로 오도할 자리였음.
- 9행 "쿨 뉴트럴 + 쿨로즈" → 웜 베이지
- 31행 "`--gold` 실제 색은 로즈" → 웜 카키/브라운 `#968062`
- 32행 "`--cta`(#C1467A)는 흰 카드 위에서만, bg 위 4.47 미달" → **제약 폐기**. 새 `--cta #7A6244`는 bg 위 **5.16** 통과
- 44행 아웃라인 근거에서 죽은 색 제거
- 33행 CHANGELOG와의 모순 해소

### `.btn-ghost` 테두리 수치 정정
문서 `2.19` → 실제 **1.88**. taupe `#ABA9B2`→`#BCB2A6`, bg `#F8F8FA`→`#F6F2EF` 로 바뀌며 **베이지 전환으로 오히려 나빠짐.** 감수 유지(다올 판단)하되 숫자만 정정. 통과시키려면 `--ink-faint`(3.29).

### 파비콘 = Parisienne 'N' 실제 글리프
- 지어낸 마크가 아니라 **헤더가 이미 쓰는 워드마크(`Nail.urban`, Parisienne)의 이니셜** — 헤더 이상으로 지어내는 게 없다는 게 근거.
- `fontTools` 로 로컬 원본(`fonts/fonts-main/ofl/parisienne/Parisienne-Regular.ttf`)에서 'N' 아웃라인 추출 → y축 반전·64px 뷰박스 피팅 → data-URI. **좌표 소수점 1자리 반올림으로 6.8KB → 2.4KB.**
- 색 = 배경 `--bg2 #EEEAE1` + 글리프 `--cta-dk #5E4A32`. 16px 탭에서도 형태 유지 확인.
- ⚠️ data-URI 는 웹폰트를 못 불러오므로 **글리프를 path 로 굽는 것 외엔 방법이 없음**(`font-family` 지정은 무효).
