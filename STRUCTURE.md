## 디자인 토큰 (실제 코드 기준)
- **⚠️ 이 팔레트는 레퍼(사장님 인스타 Notice 카드) 픽셀 추출 기반. 눈대중 아님.** ★ = 추출값 그대로.
- **구조가 핵심 — 균일 채도가 아니라 곡선**:
  | 구간 | 토큰 | R-B | 성격 |
  |---|---|---|---|
  | 밝은 면 (L 89~98%) | ★bg `#F6F2EF`(7) / ★bg2 `#EEEAE1`(13) / card `#FCFBFA`(2) / line-soft `#E9E4DC`(13) | ≤13 | **중성** |
  | 중간톤 (L 69~83%) | ★line `#DAD3C9`(17) / ★gold-mark `#E4D7C5`(**31**) / taupe `#BCB2A6`(22) | 17~31 | **웜 (여기에만 따뜻함)** |
  | 어두운 면 (L 19~52%) | ink-faint `#8A857D`(13) / ★ink-soft `#696661`(**8, 채도 4%**) / ink `#33312E`(5) | ≤13 | **중성** |
  | 액센트 | gold `#968062` / cta `#7A6244` / cta-dk `#5E4A32` | 44~54 | 웜 |
- **⚠️ 절대 하지 말 것: 전 밝기를 R>G>B 로 미는 것.** 그러면 노란기가 돈다(실패 경험 3회: 채도 47/32/27 전부 아님). [[palette-warm-beige-drift]]의 "전부 R=G=B로" 규칙은 **중성 방향으로만 성립**하며, 웜 방향으로 뒤집으면 안 됨.
- 특히 **잉크를 따뜻하게 하지 말 것** — 글자는 페이지 전체에 깔려서 영향이 가장 크다(내 실패: R-B 25 vs 레퍼 8 = 3배).
- `gold` 는 bg2 위 3:1 통과선이 `#968062`(3.15). `#A08A6B` 는 2.76 미달.
- ⚠️ **`--gold` 는 3:1 등급색 = 테두리·문양 전용. 텍스트에 쓰지 말 것**(card 위 3.65 = 일반 텍스트 4.5 미달). 텍스트엔 `--cta`(card 5.55). 실제로 STEP 라벨에 썼다가 걸림.
- **파비콘 = Parisienne 'N' 실제 글리프**(헤더 워드마크와 동일 폰트에서 fontTools 로 벡터 추출 → data-URI, 2.4KB). 배경 `--bg2` + 글리프 `--cta-dk`. **지어낸 마크가 아니라 이미 쓰는 워드마크의 이니셜**이라는 게 근거. ⚠️ data-URI 라 hex 가 URL 인코딩됨 → 색 검사 시 `unquote()` 필수.
- 브랜드 마크는 예외(카톡 `#FEE500`·`#3B1E1E`, 인스타 그라데이션)
- ⚠️ 변수명 `--gold`는 3톤 공통 find-replace 편의로 유지(실제 색은 **웜 카키/브라운 `#968062`**. 로즈 아님 — 옛 설명 잔재였음)
- **`--cta` `#7A6244` 는 bg 위 5.16 / card 위 5.55 로 통과** — 로즈 시절의 "흰 카드 위에서만" 제약은 **폐기**(그 근거였던 `#C1467A` 4.47 미달은 팔레트 교체로 소멸). 다만 **아웃라인·사진 위에는 여전히 `--cta-dk`** 를 쓸 것. → [[outline-button-aa-trap]]
- ⚠️ **알려진 미달 1건(감수 판단, 2026-07-15 전수 재감사 기준)**: `.btn-ghost` 테두리 taupe `#BCB2A6` — bg 위 **1.88:1**(1.4.11 UI 3:1 미달). 베이지 전환으로 오히려 나빠짐(로즈 시절 2.19). 통과시키려면 `--ink-faint`(3.29). 레이블 텍스트가 고대비라 감수 판단.
- ✅ `.tagline` 은 **해결됨** — `--cta-dk` 로 교체(사진+스크림 위 미달 80%→**0%**). 로즈 시절엔 cta-dk 로도 31% 미달이라 "해결 불가"였으나 **베이지 팔레트에선 성립하지 않음**.
- ⚠️ **사진 위 텍스트는 일반 대비 스캔으로 못 잡는다** — 부모를 타고 올라가 불투명 배경색을 찾는 방식은 사진을 못 봄. **캔버스로 이미지 픽셀을 떠서 스크림 알파와 합성**해야 함(`.tagline` 검사 방식).
- ⚠️ **웜 hex 예외 2개 = 브랜드 마크**: `#FEE500`(카톡), 인스타 로고 그라데이션(#FFDD55·#FF543E·#C837AB). 팔레트 스캔에서 무시할 것.

## 레이아웃 규칙 (아뜰리에 실측 기반)
- **border-radius 전부 0** (버튼·카드·사진·지도·입력창)
- **box-shadow 0 · 호버 들림(translateY) 없음** = 플랫
- 사진: about 1:1 정사각 / 갤러리 = 모자이크
- **모자이크 갤러리**: 데스크톱 4열×3행(aspect 4/3) — 큰 정사각(2×2) 1 + 가로형(2×1) 3 + 작은 정사각(1×1) 2 = 12셀 정확히 채움(빈틈 0). 모바일 2열×5행.
- 시그니처 카드: 아이보리 배경 + gold 테두리 + 배지 (그라데이션 없음)
- **예약 섹션 = 카톡 CTA + QR(PC전용) → STEP 1·2·3 → 예약 안내.** **폼 없음**(사장님이 카톡으로만 받으심 — 폼을 두면 고객이 남기고 답을 못 받는 구조가 됨). `.res-grid`·`.chan`·`.form*` 전부 삭제됨.
  - **QR = `nu-kko-qr.svg`(self-host)**. PC 전용(`@media min-width:720px`) — 폰은 버튼을 누르면 되고, PC 는 카톡 링크가 로그인 벽. ⚠️ **콰이어트 존 4모듈 필수**(ISO/IEC 18004). `qrcode` 라이브러리 기본 `border=4` 를 줄이지 말 것 — `border=2` 로 만들었다가 규격 미달로 걸림.
  - **STEP 1·2·3 = 사장님 인스타 실제 내용**(오마카세 *디자인 과정*). ⚠️ 지어내지 말 것 — 내가 *예약 절차*로 지어냈다가 방향이 통째로 틀렸음.
  - ⚠️ **CTA 는 섹션 상단 고정**(스텝 뒤로 옮기는 깔때기 안을 2회 시도했으나 흐름이 어색해 롤백). PC 에서 CTA 줄이 44%만 차서 허전한 건 **감수 판단**.
- **버튼 = 아웃라인(에디토리얼)**: 흰 .72 backing + 1px `--cta-dk` 테두리 + `--cta-dk` 텍스트, 자간 .14em / 웨이트 500 / 패딩 18·44. 호버 = `::before` 좌→우 fill sweep + 흰 텍스트. `.btn`·`.nav-cta`·`.map-btn` 동일 규칙. ghost = taupe 테두리 + ink 텍스트.
  ⚠️ 버튼색은 `--cta-dk`(bg 위 7.54). 아웃라인은 액센트가 *배경→텍스트*로 바뀌어 대비 기준이 뒤집히므로 한 톤 내리는 게 원칙. 예외 = 모바일 하단바 `.qmain`(솔리드 유지).
- **문양(deco)**: 섹션당 **4개**(`.band`만, 섹션 상하 여백 띠에 배치 → 전 폭에서 텍스트·이미지와 겹침 0). `.deco-layer`가 콘텐츠 칼럼(min(1280px,100%)) 기준이라 창 모드에서도 안 깨짐.

## 스크립트
등장 리빌(IntersectionObserver + 1.5s 강제표시 폴백) · 라이트박스(role=dialog/aria-modal, alt 승계) · 폼(데모 가짜성공, 검증+focus, role=alert, 중복제출 가드) · **앵커 부드러운 이동**
- **앵커 이동 = rAF 커스텀 700ms/easeInOutCubic** (`a[href^="#"]` 위임). CSS `scroll-behavior:smooth`는 남겨뒀지만 **OS 동작줄이기가 켜지면 무력화되므로 실질 동작은 rAF가 담당**. 루프는 `behavior:'instant'`로 CSS smooth와 충돌 방지.
- ⚠️ **`setTimeout(DUR+300)` 가드 필수** — rAF는 백그라운드 탭에서 안 돌고, preventDefault 뒤라 링크가 죽는다. 초과 시 즉시 이동 폴백.
- 휠·터치 시 양보 / 해시는 replaceState / 도착 섹션에 tabindex=-1 + focus()
- 오프셋 = 헤더 실측 높이 + 10px (헤더 65px → 착지 74~75px)
