## 디자인 토큰 (실제 코드 기준)
- **⚠️ 이 팔레트는 레퍼(사장님 인스타 Notice 카드) 픽셀 추출 기반. 눈대중 아님.** ★ = 추출값 그대로.

  | 밝은 면 (L 89~98%) | ★bg `#F6F2EF`(7) / ★bg2 `#EEEAE1`(13) / card `#FCFBFA`(2) / line-soft `#E9E4DC`(13) | ≤13 | **중성** |

  | 어두운 면 (L 19~52%) | ink-faint `#8A857D`(13) / ★ink-soft `#696661`(**8, 채도 4%**) / ink `#33312E`(5) | ≤13 | **중성** |
  | 액센트 | gold `#968062` / cta `#7A6244` / cta-dk `#5E4A32` | 44~54 | 웜 |
- **⚠️ 절대 하지 말 것: 전 밝기를 R>G>B 로 미는 것.** 그러면 노란기가 돈다(실패 경험 3회: 채도 47/32/27 전부 아님). [[palette-warm-beige-drift]]의 "전부 R=G=B로" 규칙은 **중성 방향으로만 성립**하며, 웜 방향으로 뒤집으면 안 됨.
- 특히 **잉크를 따뜻하게 하지 말 것** — 글자는 페이지 전체에 깔려서 영향이 가장 크다(내 실패: R-B 25 vs 레퍼 8 = 3배).
- `gold` 는 bg2 위 3:1 통과선이 `#968062`(3.15). `#A08A6B` 는 2.76 미달.
- **파비콘 없음(의도)** — 근거 없는 마크는 없는 것만 못함. 사장님 로고 확보 시 추가.
- 브랜드 마크는 예외(카톡 `#FEE500`·`#3B1E1E`, 인스타 그라데이션)
- ⚠️ 변수명 `--gold`는 3톤 공통 find-replace 편의로 유지(실제 색은 로즈)
- **`--cta`(#C1467A)는 흰 카드 위에서만 텍스트로 쓸 것**(4.75:1). `--bg`/`--bg2` 위 텍스트·아웃라인은 **`--cta-dk`**(5.75/5.23). `--cta`는 bg 위 4.47로 AA 미달. → [[outline-button-aa-trap]]
- ⚠️ **알려진 미달 2건(감수 판단)**: `.tagline`(사진+스크림 위, 98% 미달 — cta-dk로도 해결 안 됨) / `.btn-ghost` 테두리 taupe 2.19:1(1.4.11 미달)
- ⚠️ **웜 hex 예외 2개 = 브랜드 마크**: `#FEE500`(카톡), 인스타 로고 그라데이션(#FFDD55·#FF543E·#C837AB). 팔레트 스캔에서 무시할 것.

## 레이아웃 규칙 (아뜰리에 실측 기반)
- **border-radius 전부 0** (버튼·카드·사진·지도·입력창)
- **box-shadow 0 · 호버 들림(translateY) 없음** = 플랫
- 사진: about 1:1 정사각 / 갤러리 = 모자이크
- **모자이크 갤러리**: 데스크톱 4열×3행(aspect 4/3) — 큰 정사각(2×2) 1 + 가로형(2×1) 3 + 작은 정사각(1×1) 2 = 12셀 정확히 채움(빈틈 0). 모바일 2열×5행.
- 시그니처 카드: 아이보리 배경 + gold 테두리 + 배지 (그라데이션 없음)
- **예약 섹션 = 카톡 버튼 하나(`.res-cta`)** + 힌트 한 줄 + 예약 안내. **폼 없음**(사장님이 카톡으로만 받으심 — 폼을 두면 고객이 남기고 답을 못 받는 구조가 됨). `.res-grid`·`.chan`·`.form*` 은 전부 삭제됨.
- **버튼 = 아웃라인(에디토리얼)**: 흰 .72 backing + 1px `--cta-dk` 테두리 + `--cta-dk` 텍스트, 자간 .14em / 웨이트 500 / 패딩 18·44. 호버 = `::before` 좌→우 fill sweep + 흰 텍스트. `.btn`·`.nav-cta`·`.map-btn` 동일 규칙. ghost = taupe 테두리 + ink 텍스트.
  ⚠️ 버튼색은 `--cta`가 아닌 **`--cta-dk`** — 아웃라인은 로즈가 *텍스트*가 되므로 `--cta`(#C1467A)는 AA 미달(4.47). 예외 = 모바일 하단바 `.qmain`(솔리드 유지).
- **문양(deco)**: 섹션당 **4개**(`.band`만, 섹션 상하 여백 띠에 배치 → 전 폭에서 텍스트·이미지와 겹침 0). `.deco-layer`가 콘텐츠 칼럼(min(1280px,100%)) 기준이라 창 모드에서도 안 깨짐.

## 스크립트
등장 리빌(IntersectionObserver + 1.5s 강제표시 폴백) · 라이트박스(role=dialog/aria-modal, alt 승계) · 폼(데모 가짜성공, 검증+focus, role=alert, 중복제출 가드) · **앵커 부드러운 이동**
- **앵커 이동 = rAF 커스텀 700ms/easeInOutCubic** (`a[href^="#"]` 위임). CSS `scroll-behavior:smooth`는 남겨뒀지만 **OS 동작줄이기가 켜지면 무력화되므로 실질 동작은 rAF가 담당**. 루프는 `behavior:'instant'`로 CSS smooth와 충돌 방지.
- ⚠️ **`setTimeout(DUR+300)` 가드 필수** — rAF는 백그라운드 탭에서 안 돌고, preventDefault 뒤라 링크가 죽는다. 초과 시 즉시 이동 폴백.
- 휠·터치 시 양보 / 해시는 replaceState / 도착 섹션에 tabindex=-1 + focus()
- 오프셋 = 헤더 실측 높이 + 10px (헤더 65px → 착지 74~75px)
